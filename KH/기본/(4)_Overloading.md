# 오버로딩 (Overloading)
* 단어 뜻 자체로는 과적, 과부화라는 뜻을 나타내고 있다. 
* 프로그래밍에서는 한 클래스 안에 같은 이름의 메서드를 여러개 정의가 가능할 수 있는 것을 말한다.
<br/>

> 사용 조건
* 같은 메서드 이름을 갖는다.
* 매개변수 개수 또는 타입이 달라야한다.
* 반환 타입은 사용하는데에 관계없다.
<br/>

### 메서드를 오버로딩
- 메서드를 오버로딩하는 이유:
  <br/>
  
    - 인터페이스에 일관성을 갖게하여 메소드가 어떤 기능을 갖고있는지 기억하기 쉬워져 일관성 있게 메서드를 호출 할 수 있다.
<br/>

### 생성자를 오버로딩
- 생성자를 오버로딩하는 이유:
  <br/>
  
    - 매개변수가 존재하는 생성자만 정의하는 경우 매개변수가 없는 생성자는 호출하지 못하는 불편한 경우가 있다. 이때 오버로딩하면 모두 정의 할 수 있어 외부에서
    객체의 생성을 상황에 따라 구분하는 것이 가능하다.
