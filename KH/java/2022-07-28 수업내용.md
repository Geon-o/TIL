# 📢 Collection
- 특정 집합체를 다루는 것을 의미
    - 특정 집합체를 다루는 가장 기본적인 형태 -> 배열
        - 배열의 단점
            - 배열의 길이가 고정
            - 한가지 타입만 저장가능
            - 중간에 값을 지정할 순 있지만 기존 내용을 전부 이동시켜야 함
<br>

- Collection
    - sort() 메소드 사용시 주의
        - 자바는 String에 대한 처리를 서포트하지만, 커스텀 자료타입인 class를 처리하려면 부가적인 기능이 필요하다.
        - collections가 기본적으로 comparable을 extens하고있어 실재 comparable은 인터페이스라 볼 수 있고,
        - 해당 인터페이스는 compareTo 프로토타입을 가지고 있어 정렬하고자 하는 클래스에 이에 대한 구현체가 필요하다.
    - 오늘 코드를 배우면서 
        - compareTo가 구현된 부분에서 결국 return으로 전달 된 값은 내부 compareTo로 전달되고 compareTo내부에서 return 값으로 해당 인덱스 위치를 바꿔주는 것

<br>

### 📢 Collection을 사용하는 이유
- 배열 중간에 값을 에어하기가 편해진다. 즉 효율성을 높여준다 -> collection을 쓰는이유!

<br>

### 🚨 프로그램 설계시 주의
- 프로그램 설계시 각각의 기능(클래스, 메소드)은 단방향으로 의존되어야한다.
    - 클래스나 메소드가 복잡하게 얽혀있다면 코드 수정에 많은 어려움을 느낄거고 즉 유지보수하기가 힘들어질 것이다.
