# problem
- n명의 사용자가 사용할 수 있고 총 상금 100억이 걸린 로또 게임이다
- 로또를 받을 때 본인의 숫자는 중복되지 않아야하고, 다른사람과 본인의 로또번호 자리수에 같은 숫자가 들어가지 않게 한다.
- 그렇게 각자 6자리 숫자를 받고 자리수 마다 계산해야한다.
    - [ (숫자1 + 숫자2 + 숫자3 + 숫자4) * 숫자5 ] / 숫자6
- 계산된 수에따라 가장 큰 값을 가진 사람이 이긴다.
- 다만 계산된 값이 동점이라면 동점자 만큼 상금을 분할하여 가져간다.

# Review
- 전체적으로 코드가 지저분하고 배열을 사용한게 아쉽다.(배열의 사용성에대해 조금 더 익숙해져야한다.)
- 배열말고 List를 사용하여 코드를 조금 더 유연하게 할 필요가 있다.
- 위의 problem에서 거짐 어느정도 구현은 하였지만 로또번호가 다른사람과 곁칠 수 있는 이슈가 있다.🚨
- 사용자들의 총 점은 모두 계산되어 젤 높은 값을 가진 사람은 구현하였으나 동점자에 대한 기능은 구현하지 못했다. 🚨
- Collections를 사용하여 조금 더 매끄럽게 결과를 다룰 필요가 있다.(사용하지 않으면 코드가 지저분해짐 -> 유지보수 힘들어짐)


# Solution
- 유저 스토리 기반으로 설계를 하다보니 클래스가 기능을 나타내고 있다.
- 클래스 이름을 최대한 기능의 전체를 나타내게하고, 세세한 기능들은 메소드로 나타내기-> 생성된 데이터는 그 클래스에서 관리하도록 설계
