# 클래스 멤버와 인스턴스 멤버
### 📍클래스 내에서 필드변수, 메서드, 생성자와 같은 멤버를 사용한다. 그 안에서 변수와 메서드의 멤버에 대해 2가지로 나눌 수 있다.
<br/>

- Static
    - Static이 붙은 변수   -> 클래스 변수
    - Static이 붙은 메서드 -> 클래스 메서드 
#### 📍 변수, 메서드를 멤버라 하는데 Static이 붙어 클래스 멤버라고 칭한다.
<br/>

- 특징
    - 클래스 멤버는 객체를 생성하지 않고 언제든지 사용가능하다.
    - 객체 생성이 불가능하며 공통속성을 갖는다.
    - this를 사용할 수 없다.
<br/>

- 형식
```java
public class Example{
  static int Testname; //클래스 변수
}

public static ExampleName(){ //클래스 메서드
 ...
}
```
<br/>

- 사용형식
```java
Example.Testname = Value;
Example.ExampleName(parameter);
```
#### 📍 이와같은 형식으로 멤버를 정의해주고 사용한다. 
<br/>

- 인스턴스 멤버
    - Static이 붙지 않은 변수   -> 인스턴스 변수
    - Static이 붙지 않은 메서드 -> 인스턴스 메서드
#### 📍 이처럼 Static이 붙지 않은 멤버는 인스턴스 멤버라고 한다.
<br/>

- 특징
    - 객체를 생성해야 인스턴스 멤버를 사용할 수 있다.
    - 객체 생성 시 인스턴스 멤버는 객체별로 멤버를 생성해야만 한다.
    - 객체를 생성하므로 개별속성을 갖을 때 유용하다.
<br/>

#### 🚨 클래스 멤버는 객체를 생성하지 못해 인스턴스 멤버를 참조하지 못하며 인스턴스 멤버는 클래스 멤버를 참조할 수 있다. 🚨
#### 🚨 결국 둘의 차이점은 인스턴스 멤버를 사용하지는지, 개별속성 이냐 공통속성 이냐의 차이이다. 🚨
