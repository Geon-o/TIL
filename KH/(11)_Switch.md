# switch

### 📍 기존의 switch
 - if문의 조건식의 따라 경우의 수가 많아질수록 else-if문을 계속 추가해야 하므로 조건식이 많아져 복잡해지고 처리시간도 많이 걸린다.
 - 이러한 점을 보완하기 위해 switch문을 사용하여 단 하나의 조건식으로 많은 경우의 수를 처리할 수 있어 한결 간결해져 알아보기도 쉬워진다.
 ```java
 switch(조건식){
  case 값1:
   ... //조건식의 결과에 따라 같을 경우 수행될 문장
   break;
   
  case 값2:
   ... // 조건식의 결과에 따라 같을 경우 수행될 문장
   break; 
 }
```
 - 위와 같은 예제로 사용된다. 하지만 기존의 switch문은 치명적인 단점이 있다. case문 안에 무조건 break값이 들어가 있어야 원하는 값이 나온다.
 - 실수로 break문을 빼고 사용하면 해당 case문을 빠져나오지 못해 기대값과 다른 값이 같이나오게 된다.
 <br/>
 
### 💡 새로운 switch
 - 점점 간결해지며 심플한 코드를 사용하길 원하는 개발자들을 위해 Java 14버전에서는 개서된 switch문을 제시했다.
    - 기존의 switch - case 형식으로 사용되지만 람다식을 이용하여 내용을 처리하게 된다.
    ```java
    switch(조건식){
      case 1 -> ... ;// 조건식의 결과에 따라 같을 경우 수행될 문장
      case 2 -> ...;
      case 3 -> ...;
      ...
    }
    ```
    - 위와 같이 람다식을 사용하여 간단한 형태로 사용할 수 있다.
    - 사실 기존의 switch문에서도 위와 같은 예제 형태로 사용하였다.
        - 람다식 대신 "yield"라는 형태로 사용하였다.
        ```java
        switch(조건식){
          case1 : yield "1";
          case2 : yield "2";
        ```
    <br/>
    
### 📍 default값
 - switch문에서 사용되는 키워드이다. 조건식의 결과에서 원하는 값이 들어오지 않을 경우 default문을 실행하고 종료하게 된다.
 - switch식의 경우 조건식의 값이 정수 혹은 문자열일 경우 default값을 필수로 기재해주어야 한다. 
    - 이유는 case문을 모든 경우에 대해 만들 수 없기 때문이다.
