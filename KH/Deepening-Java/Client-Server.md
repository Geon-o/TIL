# Client & Server

### Client와 Server의 네트워크를 통해 통신하는 프로그램을 만들면서 알게된 것

- close()메서드
    - close()메서드의 사용이 중요한 것 같다. 
    - 보통 close()메서드가 있는 클래스는 OS(운영체제)부터 메모리를 할당받게되어 외부자원을 사용하고면 더 이상 사용하지 않는다는 신호로 close()메소드를 사용해야한다.
        - close()메소드를 사용하지 않는다면 메모리는 계속 할당되어 문제가 될 것이다.
<br/>

- try-with-resource
    - 간단한 통신 프로그램이라면 문제가 없을 것이다. 하지만 다수통신을 하는 프로그램 Server에서 예외가 발생해 close메소드를 호출하지 못하고 종료가 되버리면 골란할 것이다.
    - 이로인해 항상 close메소드를 호출해주는 구문으로 try-with-resourse를 사용해준다.
    ```java
    try(close가 필요한 객체 변수){ // () 안에 resource 객체 구문을 넣어줌
      처리내용
    }
    ```
    - 이와 같이 필요한 부분에 입력해주면 되겠다.
    - try블록이 정상적으로 실행을 완료했거나, 도중에 예외가 발생하면 자동으로 resource의 close()메소드가 호출된다.
        - try블록에서 예외가 발생해 빠져나오면 catch문으로 예외를 처리가 가능하다.
