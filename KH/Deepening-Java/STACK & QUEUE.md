# Stack & Queue

### 💡 Stack
 - 데이터를 차곡차곡 쌓은 상태에서 가장 마지막에 넣은 것을 먼저 꺼내는 방식이다.
 - Stack은 같은 구조와 크기의 자료를 정해진 방향으로만 쌓을 수 있다.
 - top이라하여 가장 최근에 들어온 데이터를 가리키고 있으며 push(넣기) / pop(꺼내기) 를 사용하여 제어할 수 있다.
 - 이러한 구조를 후입선출 LIFO구조라고 한다.
<br/>

### 💡 Queue
 - 터널 형태의 앞, 뒤가 열려있는 형태를 띄고 있으며 한쪽 끝에서는 삽입 작업(push), 다른쪽에서는 삭제 작업(pop)이 이루어진다.
 - 삽입 작업만 하는 곳을 front라 하며, 삭제 작업하는 곳은 rear라 한다.
 - 이러한 형태는 처음들어간 데이터를 가장 먼저 꺼내는 방식으로 선입선출 FIFO구조라고 한다.
<br/>

### 💡 DFS(깊이 우선 탐색)
 - 최대한 깊이 내려간 뒤 더 이상 갈 곳이 없으면 옆으로 이동한다.
 - 모든 노드를 방문하면서 진행하기 떄문에 검색 속도는 다소 느리다.
 - Stack을 사용하여 구현이 가능하다.
<br/>

### 💡 BFS(너비 우선 탐색)
 - 최대한 넓게 이동한 뒤 더 이상 갈 곳이 없으면 아래로 이동한다.
 - 루트 노드(시작점)에서 가장 인접한 노드로 이동하기 때문에 검색속도가 빠르며 최단 경로를 알 수 있다.
 - Queue를 사용하여 구현이 가능하다.
<br/>

#### ✏ 프로그램에서는 BFS보다 DFS를 많이 사용하는 경우가 있다
