# Exception

## 📍 
  - 프로그램 실행 흐름상 오류가 발생했을 때 오류를 그대로 실행시키지 않고 오류에 대응하는 방법을 제시하는 개념이나 하드웨어 구조를 의미하는게 예외처리이다.
  - 프로그램에서 실행 중 특정 문제가 발생하면 발생하는 부분을 처리하여 흐름을 바꾸는 개념이다.
<br/>

## ❓ 오류(Error)와 예외(Exception)의 개념
- 오류(Error)
    - 프로그램에서 처리할 수 없는 치명적인 문제를 의미한다.
    - 개발자가 미리 예측하여 처리할 수 없다.
<br/>

- 예외(Exception)
    - 개발자가 구현한 프로그램에서 발생한 실수나 사용자의 영향에 의해 발생한다.
    - 이는 개발자가 미리 예측하여 처리할 수 있다.
<br/>

## 💡
- 예외처리
    - try / catch문 사용
    - catch문은 여러번 사용가능하다. 즉 발생하는 Exception을 여러가지 잡아 처리할 수 있다.
```java
public class Test {

  try {
  
    구현부분...
    
  } catch(예외처리문) {
  
    예외처리 실행부분...
    
  } finally {
    ...
  }
  ```
- finally
    - try 블럭 내에서 예외의 발생 유무에 관계없이 반드시 처리해준다.
    - 무조건 사용되는 키워드는 아니기때문에 생략이 가능하다.
<br/>

## ✏
  - 예외처리를 학습하면서 막히는 부분은 try 블럭과 catch 블럭의 사용하는 위치나 처리하는 방법이 미숙하다 느껴졌다. 
  프로그램을 돌리면서 이러한 예외상황에 대처 능력을 능숙하게 사용하기 위해 많은 경험이 필요하다고 느껴졌다.
