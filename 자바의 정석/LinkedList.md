# LinkedList

### 배열의 단점을 보완한 것이다.
<br/>

### 🏷 배열의 장/단점
- 장점: 구조가 간단하여 데이터를 읽는 시간이 짧다. 즉 접근시간이 빠르다.
- 단점: 한번 생성 시 크기 변경이 불가능하다. 크기 변경을 하려면 새로운 배열을 생성 후 기존 데이터를 복사해야한다.
- 또 비순차적인 데이터 추가/삭제 시 시간이 오래걸린다. 반면 순차적 추가/삭제는 빠르다.
<br/>

### 💡 배열의 단점 보완
- LinkedList
    - 배열과 달리 불연속적으로 존재하는 데이터를 연결하는데 유용하다.
    - 데이터를 삭제할 시 요소의 다음 요소 참조 값만 변경하면 된다. 또 추가 할 시에 요소 객체생성 후 참조변경만 하면 가능하다.
    <br/>
    
    - 하지만 단점도 존재한다.
        - 접근성이 떨어진다. 즉 불연속적인 형태라 요소의 다음 요소로 이동하는 건 가능하지만 위치로부터 다음요소를 건너뛰고 그 다음 요소로 이동할때 한번에 이동할 수가 없다.
    <br/>
    
- 이중 연결 리스트
    - 접근성이 떨어지는 단점을 보완하였다. 
    - 다음 요소와 이전요소를 참조하므로써 앞, 뒤로 이동이 가능해졌다.
    - 하지만 이중 연결 리스트도 단점이 존재한다. 앞,뒤로는 이동이 가능하지만 요소를 건너뛰어 이동하는 것은 불가능하다.
<br/>

- 이중 연결 원형 리스트
    - 맨 앞과 맨 끝의 값 null을 활용하여 마지막 요소의 다음을 맨 앞을 가리키고, 앞의 요소는 맨 끝을 가리키게한다.

<br/>

### 💡 ArrayList와 LinkedList 성능비교
- 순차적 추가,삭제는 ArrayList가 빠르다.
- 비순차적 추가.삭제는 LinkedList가 빠르다.
- 접근시간은 ArrayList가 빠르다.
