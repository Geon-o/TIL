# Collection framework

### collection
- 여러 객체를 모아 놓은 것이라 정의한다.
### framework
- 표준화, 정형화 된 프로그래밍 방식이라 정의한다.
    - ❗ 라이브러리와 같은 의미로 볼 수 있지만 라이브러리는 기능만 제공한다. 하지만 framwork는 기능은 물론 프로그래밍 방식까지 제공하므로 프로그램의 생산성이 향상되고 그에 따라 유지보수하기 수월해졌다.
<br/>

#### 💡 collection framework
- 다수의 객체를 다루기 위해 표준화 된 프로그래밍 방식이라 정의할 수 있다.
- 다수의 데이터를 쉽고 편리하게 다룰 수 있게 클래스를 제공한다.(저장, 삭제, 정렬 등등...)
- java.util 패키지에 포함되어 있다.

<br/>

#### 💡 핵심 인터페이스
- 크게 3가지로 구분한다.
    1. List
        - 데이터의 저장 순서가 있고, 중복을 허용한다.
    2. Set
        - 데이터의 저장 순서가 없고, 중복을 허용하지 않는다.
    3. Map
        - 키와 값이 쌍으로 이루어진 데이터 집합이며, 데이터 저장 순서가 없고,
        - 키는 중복을 허용하지 않지만, 값은 중복을 허용한다.
        - ex) id 와 password

<br/>

#### ❗ List와 Set의 공통부분만 합하여 collection 인터페이스를 따로 정의하였다.
