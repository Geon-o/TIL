# StringBuilder

### StringBuffer클래스와 비슷하지만 동기화가 안되어있다.
<br/>

### 💡 StringBuilder와 StringBuffer의 차이점에대해 알아보았다.

- 쓰레드
    - 싱글 쓰레드 (StringBuilder)
        - 한번에 1개의 작업만 가능
        <br/>
        
    - 멀티 쓰레드 (StringBuffer)
        - 한번에 여러개의 작업이 가능(동시작업)
        - 멀티 쓰레드는 데이터를 쓰레드끼리 공유하여 작업 중인 데이터에 다른 쓰레드가 접근하여 자칫 잘못하면 데이터가 꼬이게된다.
            - 이러한 멀티쓰레드의 단점을 보완하고자 동기화하는게 있다.
<br/>

- 동기화
    - 데이터를 동시에 접근하거나 변경하는 걸 막는 기법이다.
    - 이런한 기법은 위에서 설명했듯이 멀티쓰레드에서 유용하다.
    - 하지만 이러한 기법은 멀티쓰레드가 아닌 싱글쓰레드에서 사용할 시 성능저하를 잃으킨다.
        - 그리하여 싱글쓰레드에서는 StringBuilder를 사용하는게 더 효율적이다.
<br/>

### ✏ StringBuilder와 StringBuffer의 차이는 그리 크지않다. 다만 프로그램 성격에 맞춰 사용하면 좋을 듯 하다.
